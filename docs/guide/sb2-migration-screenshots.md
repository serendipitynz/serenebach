- 画像の保存先：`docs/guide/assets/sb2-migration/`（ファイル名は各図に記載）
- 本文側は現在プレースホルダ（`📷 図N`）です。撮影後、該当箇所を `![図N｜タイトル](assets/sb2-migration/figNN-....png)` に差し替えます。
- **個人情報は必ず伏せます**（[加工の共通ルール](#加工の共通ルール) 参照）。実運用データが写り込む画面（FTP・旧 SB2）では特に注意してください。

## 撮影環境の再現手順

図9〜図15（取り込み後の新 Serene Bach の画面）は、**このガイド専用のサンプルブログ** を取り込んだ状態で撮影します。実運用データ（`_sandbox/sb2/data`）は個人情報を含むため使いません。

### サンプルデータの場所と再生成

サンプル SB2 データは [`docs/guide/samples/sb2-migration/`](samples/sb2-migration/README.md) にコミットしてあります（EUC-JP・PII フリー）。そのまま取り込みに使えます。内容を変えたいときは生成スクリプトで再生成できます。

```bash
python3 docs/guide/samples/sb2-migration/make_sample.py
# → docs/guide/samples/sb2-migration/data/ が丸ごと再生成される
```

サンプルの内容（撮影時に画面へ出るはずの値）は次のとおりです。詳細は [サンプル README](samples/sb2-migration/README.md) を参照。

- **ブログ名**：山あるき日記 / 説明：週末の低山歩きと道具の記録。移行ガイド用のサンプルブログです。
- **設置 URL（旧）**：`https://example.com/blog/`（サブパス設置 — URL 転送の実演用）
- **カテゴリー（5件）**：日記 / 山あるき / 道具 / 低山 / お知らせ（「道具」「低山」は「山あるき」の子）
- **記事（12件）**：公開 **10 件**＋下書き 2 件（取り込まれるのは公開 10 件）
  - 「高尾山に登りました」（山あるき）は追記あり・保存名 `takao`・本文に画像 `img/takao.jpg`。トラックバック 2 件・コメント 3 件が付く
- **コメント（10件・すべて公開記事に付く）**：承認済み 7 件・承認待ち 3 件
- **トラックバック（3件・取り込み対象外）**：記事「高尾山に登りました」「陣馬山から高尾山へ縦走」に付く
- **テンプレート（2件）**：標準テンプレート（`{trackback}` ブロックを含む → 未対応タグ警告）／シンプルテンプレート
- **その他（取り込み対象外・SB2 の体裁用）**：利用者 2 / リンク / 画像登録 / プラグイン / Amazon。`data/` 直下のファイル構成は実運用データと一致

### 取り込み済みデータベースの用意

```bash
# 作業用の一時 DB を作り、管理者を作成（サンプル記事なし）
SB_SEED_NO_SAMPLES=1 go run ./cmd/serenebach --db=./data/shots.db seed

# サンプル SB2 を取り込む
go run ./cmd/serenebach --db=./data/shots.db import --sb-version 2 docs/guide/samples/sb2-migration/data
```

取り込み時の想定出力（図9 の内容）：

```text
import: weblog updated=true, templates=2, categories=5, entries=10, skipped=2
import: warning: template "標準テンプレート": block {trackback} not supported by Go port (trackback feature is out of scope)
```

### 撮影用サーバの起動

```bash
go run ./cmd/serenebach --db=./data/shots.db serve
# ブラウザで http://localhost:8080/ と http://localhost:8080/admin/login を開く
```

> 💡 図7（初期設定 /setup）だけは、管理者作成前の空の DB が必要です。撮影時は別途、管理者未作成の DB で起動し `http://localhost:8080/setup` を開いてください。

## 加工の共通ルール

矢印やスポットライトで「見てほしい場所」を明確にします。全図で見た目を揃えてください。

| 加工 | 用途 | 推奨スタイル |
| --- | --- | --- |
| **矢印** | クリックする場所・注目点を1つ指す | 太めの実線、色は統一（例：朱色 `#e4572e`）、先端は指す対象の少し外側 |
| **スポットライト** | 周囲を暗くして1か所を浮かせる | 周囲を 40〜60% 暗く、対象は明るいまま。数値やボタンなど「点」を見せるとき |
| **囲み枠** | 範囲（一覧の行・入力欄など）を示す | 角丸の枠線、色は矢印と統一、線は 3px 程度 |
| **番号バッジ** | 手順が複数写る画面で順序を示す | ①②③ の丸数字。左上〜右下の順で |
| **ぼかし / 黒塗り** | 個人情報・実運用値を隠す | 強めのモザイクまたは黒塗り。復元不可能な強度で |
| **トリミング** | 不要な余白・無関係な UI を除く | 注目部分が画像の主役になるように切る |

### 個人情報の取り扱い（必須）

- **氏名・メールアドレス・IP アドレス・アクセスログ・ban リスト** など、個人を特定しうる情報が写る場合は必ず **ぼかし／黒塗り** で隠します。
- 実運用データが写り込む可能性がある図（**図2・図3・図5・図8**：FTP・自分のフォルダ・OS ダイアログ）は、可能なら **サンプル用の空フォルダ／ダミー環境** で撮り直すか、写った固有名を隠してください。
- サンプルブログの内容（山あるき日記、投稿者名 さとう / たかはし / やまね など）は **架空** なので、そのまま写して構いません。

### 画像の体裁

- 形式：PNG（写真的なグラデーションが主体なら JPG も可）
- 幅：長辺 1600px 程度（Retina 相当）。本文表示は縮小されるため、文字がつぶれない解像度で。
- OS の壁紙・通知バッジ・メールアドレスなど、無関係な私物が写らないようにする。
- 撮影ツールの例：macOS は `⌘⇧5`＋プレビュー／[Skitch] など、Windows は Snipping Tool＋[ScreenToGif] など。矢印・ぼかしはこれらで付けられます。

### 図5｜Windows と macOS の警告（Windows は実行で進める／macOS はダブルクリックでは進めない）

- **目的**：OS ごとの警告の見え方と、そこからどうするかを示す。**Windows は解除して実行できる／macOS（26 など）はダブルクリックでは進めず、ターミナルを使う**、という違いを伝える。
- **画面**：**2枚並び（左：Windows、右：macOS）**。
  - Windows：SmartScreen「WindowsによってPCが保護されました」→ **詳細情報 → 実行**（①→②）。実行するとターミナルが開いて DB 初期化・`:8080` 起動まで進む様子も入れると親切。
  - macOS：Gatekeeper「"serenebach" は開けません（Apple はマルウェアでないことを確認できません）」で、ボタンが **完了（Done）／ゴミ箱に入れる（Move to Bin）** の 2 つだけ、の状態（撮影済み素材 `SCR-20260716-rtix.png`）。
- **強調**：
  - Windows 側：**詳細情報 → 実行** を番号バッジ ①②＋囲み枠。
  - macOS 側：**「完了（Done）」を囲み枠**、**「ゴミ箱に入れる」に ✕ や「押さない」注記**（誤削除防止）。「ダブルクリックでは進めない → ターミナルを使う」の吹き出しを添える。
- **加工**：2枚を横並び合成。囲み枠＋矢印＋注記。ユーザー名などが写ったら伏せる。
- **保存先**：`fig05-unblock.png`
- **備考**：macOS 26 では右クリック→開くの回避は不可。この図は「解除方法」ではなく「**この警告が出ても慌てない・Done を押してターミナルへ**」の注意喚起。実行手順そのものはガイド本文の 4-1（ターミナルで `tar` 展開→`./serenebach`）が正。

### 図9｜取り込み後のターミナル表示

- **目的**：取り込みコマンドの実行と結果（件数・警告）を見せる。
- **画面**：ターミナル／PowerShell に、入力したコマンドと出力の両方が写る状態。
  - 入力：`./serenebach import --sb-version 2 ./sb2data`
  - 出力：`import: weblog updated=true, templates=2, categories=5, entries=10, skipped=2` と `warning:` の行。
- **強調**：`entries=10` と `skipped=2` を **スポットライト**。`warning:` の行に **下線または囲み**。
- **加工**：スポットライト＋下線。吹き出しで「skipped=2 は下書き2件」「warning はテンプレの未対応タグ」を添えると親切。
- **保存先**：`fig09-import-result.png`
- **備考**：サンプルを取り込むとこの出力が再現できる。
