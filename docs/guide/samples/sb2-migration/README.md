# サンプル SB2 ブログ（移行ガイド用）

このフォルダは、[Serene Bach 2 からの移行ガイド](../../sb2-migration.ja.md) の説明・スクリーンショット・件数例に使う **架空のサンプル SB2 ブログ** です。

- **個人情報は一切含みません。** 実運用データ（`_sandbox/sb2/data`）はコメント投稿者名・メール・IP・パスワードハッシュなどを含むため使わず、このサンプルを別途合成しています。IP アドレスはすべて RFC 5737 のドキュメント用レンジ（`192.0.2.0/24` / `198.51.100.0/24` / `203.0.113.0/24`）、メール・URL は `example.com` / `example.net` です。
- `data/` 以下の `*.cgi` は **Serene Bach 2 のデータファイル**（TAB 区切りのテキスト）で、実行スクリプトではありません。SB2 がこの拡張子で記録します。
- `img/` には記事・画像ライブラリが参照する画像ファイル（`takao.jpg` / `mitake.jpg` / `jinba.jpg`）を置いています。SB2 では画像は `data/` の**外**（CGI ルート直下の `img/`、`conf_dir_img`）に置かれるため、このサンプルでも `data/` と同じ階層の `img/` に分けています。この 3 枚は位置情報などのメタデータを含まない架空の風景写真です。
- **Serene Bach 2 本体が作るファイルを一通りそろえて** あります。索引ファイル（`entry.cgi` など）や、移行では使わない SB2 のファイル（`trackback/`・`user/`・`link.cgi`・`amazon.cgi` など）も含めているのは、読者が「自分の `data/` にあってサンプルに無いファイル」で戸惑わないためです。各ファイルの意味と移行対象かどうかは、ガイドの [data フォルダの中のファイルについて](../../sb2-migration.ja.md#付録data-フォルダの中のファイルについて) を参照してください。SB2 本体とは無関係な第三者プラグイン由来のファイルは、下記のとおり除いています。
- 文字コードは **EUC-JP**。SB2 の実態に合わせてあり、インポータの文字コード自動判別も同時に確認できます（`git diff` では中身が文字化けして見えますが、内容の“正”はこの生成スクリプトです）。

## 中身

架空の登山日記「山あるき日記」です。

| 種類 | 件数 | 内容 |
| --- | --- | --- |
| ブログ | 1 | 山あるき日記（`https://example.com/blog/` にサブパス設置の想定） |
| カテゴリー | 5 | 日記 / 山あるき / 道具 / 低山 / お知らせ（「道具」「低山」は「山あるき」の子） |
| 記事 | 12 | 公開 **10** 件＋下書き 2 件（SB2 の記事ステータスは 0=下書き / 1=公開 / 2=公開+トップ表示 のみ） |
| コメント | 10 | 承認済み 7 件・承認待ち 3 件（すべて公開記事に付く） |
| トラックバック | 3 | 記事「高尾山に登りました」「陣馬山から高尾山へ縦走」に付く（**移行対象外**） |
| テンプレート | 2 | 標準テンプレート（`{trackback}` を含む → 未対応タグ警告）／シンプルテンプレート |
| 利用者 | 2 | sanpo / yamane（**移行対象外**・パスワードは架空のダミー） |
| リンク・画像・プラグイン・Amazon | 各数件 | いずれも SB2 としての体裁用（**移行対象外**） |

記事「高尾山に登りました」は、追記あり・保存名 `takao`・本文に画像 `img/takao.jpg` を含み、URL 転送や画像の扱いの説明に使えます。画像の実ファイルは上記 `img/` にあります（Go 版インポータは画像ファイルを取り込まないため、移行後は手動コピーするまでリンク切れになる、という説明にも使えます）。

## 試しに取り込んでみる

```bash
# 取り込み先を用意（サンプル記事なし）
SB_SEED_NO_SAMPLES=1 ./serenebach --db=./try.db seed

# このサンプルを取り込む
./serenebach --db=./try.db import --sb-version 2 docs/guide/samples/sb2-migration/data
```

想定される出力（ガイドのステップ5-3 と同じ）:

```text
import: weblog updated=true, templates=2, categories=5, entries=10, skipped=2
import: warning: template "標準テンプレート": block {trackback} not supported by Go port (trackback feature is out of scope)
```

- `entries=10` … 公開記事 10 件。
- `skipped=2` … 下書き 2 件（公開済みのみ取り込む仕様）。
- `templates=2` … いずれも無効状態で取り込み。
- `warning` … 標準テンプレートの `{trackback}` が未対応（トラックバック非対応）。

コメント 10 件はすべて公開記事に付いているので全件取り込まれます。トラックバック 3 件はインポータが読み取らない（対象外）ため、出力にも現れません。

## 実データとの違い（あえて省いたもの）

- **`ping.cgi` / `sbforum_rss.cgi` / `sb_sforum_rss.cgi` / `sb_pforum_rss.cgi`**：実運用データ（`_sandbox/sb2/data`）には入っていますが、これらは **Serene Bach 2 本体のデータではなく**、ping コミュニティや第三者プラグインが作るキャッシュです（SB2 に `Ping` / `Forum` のデータクラスは無く、`id.cgi` にもカウンタがありません）。SB2 のサンプルとしては不適切なので含めていません。
- **`init.cgi`**：URL 形式（`basic_preid` / `basic_suffix` など）の設定ファイル。SB2 では `data/` ではなく **CGI ルート（`sb.cgi` と同じ場所）** にあるため、`data/` をダウンロードしても含まれません。このサンプルも同じく含めていません。標準の `eid` / `.html` 形式であれば、無くても `configure.cgi` だけで旧 URL 転送は動きます。
- **`log/`（空フォルダ）**：アクセスログ用のフォルダ。空フォルダは Git で追跡できないため、生成スクリプト実行時に作られます。

## 再生成する

内容を変えたいときは、生成スクリプトを編集して実行します。`data/` が丸ごと再生成されます（`img/` の画像は生成対象外で、直接コミットしています）。

```bash
python3 docs/guide/samples/sb2-migration/make_sample.py
```

フィールドの並び順は SB2 の Perl ソース（`_sandbox/sb2/lib/sb/Data/*.pm` の `elements()` と `Driver/Text.pm` の `%ContentOfList`）に基づいてスクリプト内にコメントしてあります。インポータ側の読み取り順は `internal/importer/sb2.go` を参照してください。
