# ガイド（非技術者向け手引き）

このフォルダには、専門知識がなくても手順どおりに進められる **作業ガイド** を置きます。ふだん FTP ソフトとブラウザだけで運用している方が、迷わず最後まで到達できることを目標にしています。

## 他のドキュメントとの違い

| 置き場所 | 位置づけ | 想定読者 |
| --- | --- | --- |
| `docs/guide/`（ここ） | **手取り足取りのチュートリアル**。スクリーンショット中心 | 非技術者・はじめての方 |
| [`docs/help/`](../help/) | 管理画面に同梱される **操作ヘルプ**（機能ごとのリファレンス） | 運用中の利用者 |
| `docs/*.md`（[architecture](../architecture.md) など） | **技術リファレンス**。仕組み・設定・内部仕様 | 開発者・上級運用者 |

同じ話題（例：SB2 からの移行）でも、ここでは「手を動かす順番」を、`docs/help` と `docs/*.md` では「仕組みと網羅的な仕様」を扱います。重複ではなく役割分担です。

## 執筆時の約束ごと

新しいガイドを追加するときは、次の規約に従います。

### ファイル名

- 本体：`<トピック>.<言語>.md`（例：`sb2-migration.ja.md`、将来の英語版は `sb2-migration.en.md`）
- 撮影指示書：`<トピック>-screenshots.md`

### 画像の置き場所

- `docs/guide/assets/<トピック>/figNN-説明.png`（例：`assets/sb2-migration/fig02-ftp-download.png`）
- 本文では、撮影前は `📷 図N` のプレースホルダを置き、撮影後に `![図N｜タイトル](assets/<トピック>/figNN-....png)` へ差し替えます。
- 概念図・フロー図など、写真ではなく図解で足りるものは **mermaid で本文に直接書く**（GitHub がそのまま描画します）。画像ファイルにしないので差分でレビューでき、修正も容易です（例：SB2 移行ガイドの図1）。

### サンプルデータの置き場所

- `docs/guide/samples/<トピック>/`（例：`samples/sb2-migration/`）にコミットします。
- 生成スクリプト（`make_sample.py` など）と生成物、内容を説明する `README.md` を同梱し、誰でも再現・確認できるようにします。

### 文体・方針

- 対象読者を冒頭で明示し、**対象外**（例：スマホのみ）も正直に書く。
- コマンド操作が避けられない場合は、**それを隠さず**、Windows / macOS 別にそのままコピーできる形で示す。
- サンプルは **個人情報を含まない架空データ** を使い、`docs/guide/samples/<トピック>/` にコミットする。実運用データ（`_sandbox/sb2/data` など）はそのまま載せない。
