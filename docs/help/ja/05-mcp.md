# AI 連携と MCP

Serene Bach は MCP に対応しています。MCP に対応したエディタや AI エージェントから、記事の検索、下書き作成、画像アップロードなどを行えます。

MCP は便利な機能ですが、書き込み権限を持つトークンを渡すと、外部ツールが記事を作成・更新できるようになります。必要な相手にだけ、必要な権限で発行してください。

## できること

読み取り権限では、公開記事、カテゴリー、タグ、画像、アクセス解析の一部を参照できます。

書き込み権限では、下書き記事の作成、記事の更新、公開、画像アップロードができます。AI に本文を直接公開させるのではなく、まず下書きとして作成し、管理画面で確認してから公開する運用をおすすめします。

## HTTP 接続

リモートの AI ツールから接続する場合は、管理画面の「設定」>「AI 設定」で MCP アクセストークンを発行します。

発行時に次の内容を指定します。

| 項目 | 内容 |
|---|---|
| 名前 | 自分が後から識別しやすい名前 |
| 権限 | 読み取り、または書き込み |
| 代理ユーザー | AI が作成・更新した記事の投稿者 |

トークン本体は発行時にだけ表示されます。後から再表示できないため、発行したらすぐに接続先ツールへ設定してください。

接続先 URL は次の形です。

```text
https://example.com/mcp
```

認証には Bearer トークンを使います。

```bash
curl -X POST https://example.com/mcp \
  -H "Authorization: Bearer sb_mcp_..." \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize"}'
```

## stdio 接続

同じサーバ上のエディタやローカルツールから使う場合は、stdio 方式で起動できます。

```bash
./serenebach --db=/path/to/blog.db mcp serve
```

Claude Desktop などでは、コマンドとして Serene Bach のバイナリと `mcp serve` を登録します。

```json
{
  "mcpServers": {
    "serenebach": {
      "command": "/path/to/serenebach",
      "args": ["--db=/path/to/blog.db", "mcp", "serve"]
    }
  }
}
```

stdio 接続は、そのコマンドを実行できるローカル環境を信頼する前提の接続方法です。共有サーバや複数人で使う端末では、実行権限や DB ファイルの扱いに注意してください。

## AI 執筆補助

MCP とは別に、管理画面内の AI 執筆補助も用意されています。利用するには `SB_AI_SECRET` を設定し、各ユーザーが「設定」>「AI 設定」でプロバイダや API キーを登録します。

記事編集画面では、本文の書き直し、続きの作成、要約、タイトルやタグの提案などに使えます。画像アップロード時の alt テキスト生成にも利用できます。

## トークンの失効

不要になったトークンは「AI 設定」から失効できます。失効後、そのトークンでの MCP 接続はできなくなります。
