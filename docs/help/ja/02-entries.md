# 記事の作成と管理

記事は「新規記事」から作成し、「記事一覧」から編集します。Serene Bach では、古い HTML 記事をそのまま残しながら、新しい記事だけ Markdown で書くこともできます。

## 本文フォーマット

記事ごとに本文の書き方を選べます。

| フォーマット | 内容 | 向いている用途 |
|---|---|---|
| HTML | 入力した HTML をそのまま表示 | 既存記事の移行、細かなマークアップ |
| Markdown | 見出し、リンク、表などを Markdown で記述 | 通常の記事作成 |
| sbtext | SB3 の簡易記法の一部に対応 | SB3 から取り込んだ記事の表示 |

HTML は自由度が高い反面、書いたタグがそのまま公開ページに出ます。共同運用で信頼できないユーザーに記事を書かせる場合は、Markdown を使う運用をおすすめします。

## 本文と追記

本文は記事一覧や記事ページに表示される中心の内容です。

「追記」は、長い続きを個別記事ページで読ませたいときに使います。SB3 の「記事続き」に相当します。テンプレート側で追記表示用のブロックが用意されている場合、追記のある記事だけに続きが表示されます。

## 記事の主な項目

| 項目 | 内容 |
|---|---|
| タイトル | 記事一覧、記事ページ、フィード、OG カードに使われます |
| カテゴリー | 記事の大きな分類です。1 記事につき 1 つ選びます |
| タグ | 記事を横断して探すためのラベルです。複数指定できます |
| 状態 | 下書き、公開、非公開を選びます |
| 投稿日時 | 記事に表示する日時です。未来の日時も指定できます |
| スラッグ | 記事 URL に使う短い名前です |
| キーワード | テンプレートで meta keywords などに使えます。「メタ情報・検索エンジン向け」グループにあります |
| 要約 | `<meta name="description">` や OG の説明文に使う要約です（`{entry_excerpt}`）。空のときは本文から自動生成（200 字まで）し、本文も空ならタイトルを使います |
| canonical URL | クロスポスト・転載元など、別の URL を正規 URL として指定したいときに入力します。`http(s)://` で始まる絶対 URL のみ。空なら出力されません |
| 検索エンジンに表示しない | オンにするとその記事に `<meta name="robots" content="noindex,follow">` を出し、`sitemap.xml` からも除外します。記事自体はサイト上に公開されたままです |
| OG カード背景 | SNS 共有画像の背景を記事ごとに変えたいときに使います。保存後に「OG カードを生成」ボタンで手動生成もできます |

## 記事の状態

| 状態 | 公開側での扱い |
|---|---|
| 下書き | 公開サイトには表示されません。管理画面からプレビューできます |
| 公開 | 公開サイト、フィード、必要に応じて llms.txt に掲載されます |
| 非公開 | 公開サイトには表示されません。公開済みの記事を取り下げたいときに使います |

未来の投稿日を指定した公開記事は、その日時が記事の日時として使われます。静的配信をしている場合は、反映したいタイミングで再構築してください。

## スラッグ

スラッグを設定すると、記事 URL に数値 ID ではなく任意の短い名前を使えます。

例: `my-first-post`

使える文字は小文字英数字とハイフンです。日本語の URL 名には対応していません。未設定の場合は、記事 ID を使った URL になります。

## 画像の挿入

記事編集画面の「画像を挿入」から、アップロード済み画像を本文に差し込めます。本文エリアへ画像ファイルを直接ドロップすると、アップロードと挿入をまとめて行えます。

HTML の記事では `<img>` タグ、Markdown の記事では Markdown の画像記法が挿入されます。

## 固定ページ

「固定ページ」は、記事とは独立した静的コンテンツページです。`/about` や `/privacy` のように、トップページやアーカイブ、フィードに表示されない独立したページを作れます。

固定ページの特徴:

- トップページ、カテゴリー、アーカイブ、フィードに**出ません**
- カテゴリー、タグ、コメント、追記は**ありません**
- URL パスを自由に指定できます（例: `/about`、`/service/pricing`）
- テンプレートを個別に選べます（未指定時はアクティブテンプレート）
- OG カードも記事と同様に生成できます

パスに使える文字は小文字英数字とハイフン、スラッシュのみです。`/entry` や `/admin` などのシステム予約パスは使えません。

## ピン留め

記事編集画面の「ピン留め」にチェックを入れると、その記事はトップページとカテゴリーアーカイブの先頭に固定表示されます。お知らせや特集など、目立つ位置に置きたい記事に向いています。

ピン留めの効きどころ:

- トップページ / カテゴリーアーカイブ: ピン留め記事が先頭に浮上
- 月別・タグアーカイブ、フィード (RSS / Atom): **影響なし**（通常の日付順）
- 個別記事ページ、prev / next ナビゲーション: 影響なし

複数の記事をピン留めした場合、ピン留め記事グループ内の並び順は通常と同じ投稿日時の降順で、その後に非ピン留め記事が続きます。

ページネーションについては、ピン留め記事は常に先頭側にまとまって配置されます。1 ページあたりの記事数より多くピン留めした場合は、あふれた分が 2 ページ目以降に繰り越されます。実用上は、1 ページの記事数に収まる範囲でピン留めすることをおすすめします。

テンプレート側では、ピン留め記事に `{entry_pinned}` タグ（値は `pinned` または空）と `pinned_entry` ブロックが用意されています。詳細は[テンプレート](templates)を参照してください。

## 記事一覧

「記事一覧」では、タイトル検索や状態別の絞り込みができます。タイトルをクリックすると編集画面に移動します。公開済みの記事は、公開ページを別タブで確認できます。
