# ユーザーと権限

複数のユーザーでブログを運用できます。権限は「管理」「上級」「一般」の 3 種類です。

## 権限の違い

| 権限 | できること |
|---|---|
| 管理 | すべての管理機能を使えます。ユーザー管理もできます |
| 上級 | 記事、画像、カテゴリー、リンク、テンプレート、コメントなどを管理できます。ユーザー管理はできません |
| 一般 | 記事、画像、タグ、コメント、アクセス解析、自分のプロフィールを扱えます |

一般ユーザーは、カテゴリー、リンク、テンプレート、デザイン、ユーザー管理にはアクセスできません。また、記事や画像については、自分が作成したものだけ編集・削除できる範囲があります。

## ユーザー管理

「ユーザー」画面は管理ユーザーだけが利用できます。

できること:

- ユーザーの追加
- 表示名、メールアドレス、権限の変更
- パスワードの変更
- プロフィールの編集
- ユーザーの削除
- 表示順の変更

ログイン名は後から変更するより、必要に応じてユーザーを作り直す運用をおすすめします。

## 管理ユーザーの保護

管理ユーザーが 1 人もいなくなると管理画面を維持できないため、Serene Bach は次の操作を拒否します。

- 自分自身の削除
- 最後の管理ユーザーの削除
- 自分自身を管理ユーザーから降格する操作

管理ユーザーは常に 1 人以上必要です。

## プロフィール

各ユーザーは「プロフィール」から自分の公開プロフィールを編集できます。

| 項目 | 内容 |
|---|---|
| 表示名 | 公開ページで著者名として表示されます |
| メール | 管理用です。通常は公開されません |
| 自己紹介 | プロフィールページやテンプレート内に表示できます |
| 本文フォーマット | 自己紹介を HTML または Markdown として扱います |
| 一覧に載せる | プロフィール一覧に表示するかを選びます |

AI 執筆補助を使う場合は、プロフィール画面や AI 設定でユーザーごとの設定を行います。

## MCP の代理ユーザー

MCP トークンには「代理ユーザー」を指定します。AI エージェントが作成・更新した記事は、その代理ユーザーの投稿として記録されます。

AI 用のユーザーを別に作っておくと、後からどの記事が AI 経由で作成されたか分かりやすくなります。
