# 全文検索

Serene Bach は公開サイトに `/search?q=<検索語>` の全文検索エンドポイントを用意しています。公開済みのすべての記事のタイトル・本文・追記・キーワードがインデックス対象です。同じインデックスを管理画面の記事一覧検索と、MCP の `search_entries` ツールも共有するので、3 つの面が常に同じヒット集合を返します。

## 検索フォームをテンプレートに置く

`/search` は動的機能のため、サーバーが立っている環境でのみ機能します。静的書き出しだけのデプロイには検索フォームは既定で出ません。テンプレートのサイドバーやヘッダにフォームを置きたい場合は、次の HTML を貼り付けてください:

```html
<!-- BEGIN search_form -->
<form action="{search_url}" method="get" role="search">
  <input type="search" name="q" value="{search_query}" placeholder="検索...">
  <button type="submit">検索</button>
</form>
<!-- END search_form -->
```

- `{search_url}` は `/search` の絶対 URL に展開されます。検索が無効化されている場合は空文字列になります（下の「静的書き出し」参照）。
- `{search_query}` は検索結果ページでは直前の入力値を返すので、入力欄に検索語をそのまま再表示できます。それ以外のページでは空文字列です。
- `{search_total}` は検索結果ページでは総ヒット数、それ以外のページでは空文字列です。

`search_form` ブロックは検索フォームが無効なときに 0-stripe され、レンダリングされません。テンプレート側で条件分岐する必要はなく、上のマークアップをそのまま置いておけば自動的に表示・非表示が切り替わります。

## 検索の仕様

検索は SQLite FTS5 の trigram トークナイザを使っています。実用上のポイント:

- **3 文字以上のトークン**: FTS5 インデックス経由で部分一致します。日本語・英語の任意の部分文字列に対応します（`"東京タワー"` は連続した 4 文字を含む記事にすべてヒット）。
- **1〜2 文字のトークン**（"東京" "大阪" "会社" のような日本語の 2 文字熟語に多い）: trigram では表現できないため、本文への LIKE スキャンにフォールバックします。同じクエリ内の 3 文字以上のトークンとは AND 結合されます。
- **キーワードはスペースで区切ってください**: `機械学習 画像認識` は両方を含む記事を探します。`機械学習による画像認識の最新手法` のようにスペースなしの長い文章を 1 つで投げると、その連続部分文字列が含まれている記事しかヒットしません。
- **記号・予約語はそのままリテラル**: `node:fs` / `foo(bar)` / `A*B` / `C++` / `max-age` すべて検索可能。FTS5 の予約語 `AND` / `OR` / `NOT`、演算子 `-` などは演算子として解釈されません。
- 公開側 `/search` では非表示カテゴリの記事は除外されます。管理画面の記事一覧検索と MCP の `search_entries` は管理者向けなので非表示カテゴリも含めて検索します。
- 下書き・公開停止の記事は公開側 `/search` には出ません。
- クエリは 200 文字までです。

## 検索結果ページ

検索結果ページは現在のアクティブテンプレートの main_body を使って描画されるため、サイドバー・フッタ・ヘッダ・スタイルがそのまま継承されます。結果固有のマークアップ用に、専用のブロックが 2 つあります:

```html
<!-- BEGIN search_results -->
<h2>"{search_query}" の検索結果</h2>
<p>{search_total} 件見つかりました。</p>
<!-- END search_results -->

<!-- BEGIN search_empty -->
<p>"{search_query}" に一致する記事はありませんでした。別のキーワードでお試しください。</p>
<!-- END search_empty -->
```

- `search_results` はヒット数が 1 件以上のときだけ展開されます。
- `search_empty` はクエリが入力されたがヒット数が 0 のときだけ展開されます。
- どちらも検索ページ以外では 0-stripe されるので、共通レイアウトに置いておけます。

ヒットした記事は通常の `entry` ブロックで繰り返し描画されるため、`{entry_title}` / `{entry_permalink}` / `{entry_date}` / `{entry_description}` / `{entry_tags}` などのタグはトップやカテゴリ一覧と同じように使えます。

## 静的書き出し

`/search` は動的ルートなので、静的書き出し成果物だけを配置するモードでは 404 になります。そのまま検索フォームを出すと壊れた導線になるため、静的書き出しでは `search_form` ブロックは 0-stripe されます。動的バックエンドを併設していて同じオリジンからの GET が届く場合は、SQLite で `weblogs.static_search_form_enabled = 1` をセットすると静的成果物でも検索フォームが出るようになります（既定 OFF）。

## 索引の再構築（`reindex`）

通常運用では、記事の INSERT / UPDATE / DELETE のたびに SQLite トリガーで FTS インデックスが自動同期されます。手動で DB を編集した、トリガが落ちている、あるいは将来トークナイザを変更したなど、ズレが疑われる場合はインデックスを丸ごと再構築できます:

```bash
./serenebach reindex
```

`reindex` は何度実行しても安全で、`entries_fts` を一度空にしてから全件入れ直します。インポート後に呼ぶ必要はありません — インポータも通常の INSERT 経路を通るため、トリガで同期されます。

## 制限事項

- bm25 ランキングは未実装で、結果は新着順固定です。
- スニペットハイライトは未実装です。
- コメント本文と固定ページは検索インデックスに含まれません。
- 静的書き出し成果物の検索フォームは、動的インスタンスに到達できる環境でしか動作しません。
