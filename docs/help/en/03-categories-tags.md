# Categories and tags

Serene Bach has two ways to classify entries: **categories** and **tags**. A useful split is to think of categories as "where the entry lives" and tags as "how readers later find it".

## Differences at a glance

| Item | Categories | Tags |
|---|---|---|
| Per entry | One | Multiple |
| Typical use | Top-level sections (diary, tech notes, announcements) | Cross-cutting labels (Go, travel, reading) |
| Created | Created in the Categories screen | Auto-created when an entry is saved |
| Hierarchy | Single parent allowed | Flat, no hierarchy |
| Public pages | Per-category listing | Per-tag listing |

## Managing categories

The **Categories** screen supports adding, editing, deleting, and reordering categories.

Each category can carry a description. Templates that consume it can render the description at the top of the category listing.

A category can also pin a specific template. Use this when you want one section to look different from the rest of the site.

## Managing tags

Tags are auto-created from the entry editor's tag field. Separate multiple tags with commas.

The **Tags** screen lets you rename a tag and change its URL slug. Deleting a tag removes the tag-to-entry associations but leaves the entries themselves intact.

## How to choose between them

- One mandatory primary classification → category
- Cross-cutting label you may want to browse later → tag
- Anything that needs its own URL or template → category
- Anything ad-hoc that grows over time (seasons, event names, technologies) → tag
